2. Installation

There are a number of installation options for Rex. The easiest method is to us one of the standalone versions that use an embedded version of Jakarta Tomact [2]. The only other requirements to run this version is Java [3] and a relational database. However, Rex is developed as a Java web application, so you can use your own application server, such as WebLogic, WebSphere, or Resin, if you already have licenses for one.

2.1. Windows Installation

If you are running on Windows, and normal Windows installer is provided that installs into the Program Files directory. You need to have Java installed prior to installing Rex. The installer also adds items into the Start Menu and a Windows Service. The Service can be used to start and stop the server, like any other Windows Service. Before running Rex, you should first configure it. There are a number of items that must be setup.

2.2. Non-Windows Installation

For other platforms, such as Linux and Mac OS X, a gzipped tar file is provided. This archive does not contain any support for init scripts or startup items. The Rex server is distributed as an executable Jar file.

2.3. Web Application Installation

A version of Rex without Tomcat is available as a zip file or gzipped tar file. This distribution includes only the web application and the administration tool. An application server is required to run the web application. However, only Java Servlets are used, so you do not need any other J2EE technology, like EJB.
